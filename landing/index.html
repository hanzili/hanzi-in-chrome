get started in 2 minutes

pick your MCP client. two steps. that's it.

1

install the chrome connector

one-time setup. connects your browser to the MCP server.

install from chrome web store →

2

add the MCP server

run this in your terminal:

claude mcp add browser -- npx -y hanzi-in-chrome-mcp

or, paste this prompt into claude code and it'll set it up for you:

Install the hanzi-in-chrome-mcp MCP server so you can control my browser. The npm package is hanzi-in-chrome-mcp. Run: claude mcp add browser -- npx -y hanzi-in-chrome-mcp
3

done. ask your agent to browse.

"go to jira and create a ticket for this bug." your agent takes it from there.

how does it compare?

other browser MCPs expose primitive tools. your AI has to do the browsing itself, step by step. hanzi in chrome is different.

hanzi in chrome

you describe a task. the agent figures it out.

  • "go to gmail and star all unread" is one tool call
  • built-in agent loop. autonomous browsing, not step by step
  • your real chrome with existing logins, cookies, extensions
  • no auth setup needed
  • works with any MCP client

other browser MCPs

playwright MCP, chrome devtools MCP, claude in chrome

  • expose primitive tools: click, type, navigate, screenshot
  • your AI has to orchestrate every step itself
  • each action is a round-trip to your agent. slow and expensive
  • headless browsers break on login, CAPTCHAs, 2FA
  • great for testing public pages. not for real tasks with your accounts

bring your own credentials

no extra API keys needed. use what you already have.

MCP server

uses your existing subscription. no API key setup.

  • claude code subscription
  • codex subscription

extension standalone

use it directly from the chrome sidepanel. supports more providers.

  • claude code / codex subscription
  • anthropic, openai, google, openrouter API keys
  • self-hosted models (ollama, llama.cpp)
